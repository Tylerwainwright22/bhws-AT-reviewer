## Goal

A single Python script (`review.py`) that a staff member runs like this:

```
python review.py "Tobii Dynavox I-Series+"
```

...and gets back a plain-language structured evaluation report printed to
the terminal. The report is generated by calling Claude via the Anthropic
Python SDK.

## Implementation design for `review.py`

### 2.1 Argument handling

Use `argparse` from the standard library. Define one positional argument:
`product_name`. If the user runs the script with no arguments, argparse
prints a usage line and exits with a non-zero code.

```
python review.py "product name here"
```

No flags required for the MVP. No interactive prompt needed — the product
name is supplied on the command line so the tool is scriptable.

### 2.2 API key

Read `ANTHROPIC_API_KEY` from the environment. If it is missing, print a
clear error message ("ANTHROPIC_API_KEY is not set. Export it before
running.") and exit with code 1. Do not hard-code or embed a key.

### 2.3 Prompt construction

Build a system prompt that gives Claude the BHWS context it needs:

- BHWS is a nonprofit IDD services org in Rapid City, SD
- The audience is clinical and vocational staff, not engineers
- AT = Assistive Technology; IDD = Intellectual and Developmental
  Disabilities
- CHOICES waiver = South Dakota Medicaid waiver that can fund AT
- Vocational context = supported employment and job training for adults
  with IDD
- Plain language required throughout; avoid jargon

Build a user message that instructs Claude to evaluate a specific product
and return the six required sections (see section 2.4 below).

### 2.4 Required output sections

The prompt must instruct Claude to produce exactly these six sections in
order, with clear headings:

1. **Product Overview** — 1–2 sentences describing what the product is
   and what it does.

2. **Target Population Fit** — Does this product suit adults with IDD in
   vocational contexts? Note any cognitive load concerns, literacy
   requirements, or adaptability features.

3. **Vocational Relevance** — Map the product to employment phases where
   applicable: job exploration, job training, on-the-job support,
   advancement/independence. If it doesn't fit any phase, say so.

4. **CHOICES Waiver Compatibility** — South Dakota CHOICES waiver can
   fund AT for eligible individuals. Note whether this product is likely
   fundable, what documentation might be needed, and any known exclusions.
   Flag uncertainty if the waiver fit is unclear.

5. **Data Privacy Flags** — Note any PHI handling, cloud storage, account
   requirements, data-sharing practices, or other privacy concerns relevant
   to a nonprofit provider. Flag if the product collects sensitive data
   about end users.

6. **Overall Recommendation** — One of three values: **Explore** (strong
   fit, pursue further), **Investigate Further** (potential fit, needs
   more research), or **Pass** (poor fit or significant concerns). Follow
   the label with 2–4 sentences explaining the reasoning.

### 2.5 API call

Use `anthropic.Anthropic()` (reads key from environment automatically).
Call `client.messages.create()` with:

- `model`: `claude-opus-4-6` — most capable, appropriate for nuanced
  evaluation tasks
- `max_tokens`: `1024` — enough for a full report without waste
- `system`: the system prompt from 2.3
- `messages`: list with one user message containing the product name and
  section instructions

### 2.6 Output formatting

Print a header line:

```
========================================
BHWS AT Product Evaluation
Product: <product name>
========================================
```

Then print the raw text content from the API response. Claude will format
the six sections with the headings specified in the prompt.

Print a footer noting that the report is AI-generated and should be
reviewed by staff before use in funding or clinical decisions.

### 2.7 Error handling

- Missing API key → clear message + exit 1 (before any API call)
- `anthropic.APIConnectionError` → "Could not reach the Anthropic API.
  Check your network connection." + exit 1
- `anthropic.AuthenticationError` → "API key was rejected. Check that
  ANTHROPIC_API_KEY is correct." + exit 1
- `anthropic.RateLimitError` → "Rate limit reached. Wait a moment and
  try again." + exit 1
- Any other `anthropic.APIError` → print the error message + exit 1

## What is explicitly out of scope

- No database or file storage
- No web framework or HTTP server
- No interactive prompts (the product name is a CLI argument)
- No unit tests in the MVP (can be added later)
- No packaging (`setup.py`, `pyproject.toml`) in the MVP
- No streaming output (single blocking call is sufficient)
- No result caching
